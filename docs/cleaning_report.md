# Gapminder 데이터 전처리 리포트

## 1. 개요

Gapminder 원본 데이터는 142개국의 1952년부터 2007년까지 5년 간격 총 12개 시점을 담고 있으며, 결측치와 중복 행이 전혀 없는 깨끗한 데이터셋입니다. 전처리 후에도 행 수가 그대로 1,704개로 유지되었으며, 주로 컬럼 타입 정규화와 소수점 정리 위주의 가공이 이루어졌습니다.

| 항목 | 값 |
|------|----|
| 원본 행 수 | 1,704 |
| 전처리 후 행 수 | 1,704 |
| 제거된 행 | 0 |
| 원본 결측치 수 | 0 |
| 원본 중복 행 수 | 0 |
| 총 국가 수 | 142 |
| 연도 범위 | 1952 ~ 2007 (5년 간격) |

---

## 2. 전처리 항목

원본 데이터에 결측치와 중복이 없었기 때문에 실질적인 행 제거는 발생하지 않았습니다. 대신 이후 분석과 시각화에 용이하도록 컬럼명을 소문자로 통일하고, 연산에 적합한 타입으로 변환하였으며, 불필요한 소수점 자리를 정리하였습니다.

| # | 처리 내용 | 설명 |
|---|-----------|------|
| 1 | 결측치 제거 | `dropna()` 적용 |
| 2 | 중복 행 제거 | `drop_duplicates()` 적용 |
| 3 | 컬럼명 정규화 | 소문자 변환 + 공백 제거 |
| 4 | 타입 변환 | `year`, `pop` → `int` / `continent` → `category` |
| 5 | 소수점 정리 | `gdppercap`, `lifeexp` → 소수점 2자리 |

---

## 3. 컬럼 타입

`year`와 `pop`은 소수점이 없는 정수이므로 `int64`로 변환하여 연산 효율을 높였습니다. `continent`는 Africa, Americas, Asia, Europe, Oceania 5개의 고정된 값만 가지므로 `category` 타입으로 변환하여 메모리를 절약하고 그룹 연산 속도를 향상시켰습니다. `lifeexp`와 `gdppercap`은 연속형 수치이므로 `float64`를 유지합니다.

| 컬럼 | 타입 |
|------|------|
| `country` | `str` |
| `continent` | `str` |
| `year` | `int64` |
| `lifeexp` | `float64` |
| `pop` | `int64` |
| `gdppercap` | `float64` |
| `iso_alpha` | `str` |
| `iso_num` | `int64` |
